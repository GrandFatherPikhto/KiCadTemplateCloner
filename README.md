# KiCad Template Cloner

Инструмент командной строки для клонирования выделенных участков печатной платы в KiCad.  
Позволяет сохранять в шаблон все объекты блока (компоненты, переходные отверстия, дорожки) и переносить их в новое место с сохранением относительного расположения и цепей.

## Возможности

- **Извлечение шаблона** (`extract`) – сохраняет выделенные объекты в YAML-файл с относительными координатами.
- **Применение шаблона** (`place`) – воссоздаёт объекты в заданной точке платы.
- Сохраняет:
  - компоненты (refdes, значение, типоразмер, слой, угол поворота)
  - переходные отверстия (диаметр, сверло, слой, цепь)
  - дорожки (ширина, слой, цепь, координаты концов)
- Все координаты хранятся относительно центра масс выделенного блока.

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/GrandFatherPikhto/KiCadTemplateCloner.git
   cd KiCadTemplateCloner
   ```

2. Создайте виртуальное окружение (рекомендуется):
   ```bash
   python -m venv .venv
   source .venv/bin/activate   # Linux/Mac
   .venv\Scripts\activate      # Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## Использование

Инструмент работает через модуль `template_cloner.cli`.

### Команда `extract` – сохранение шаблона

```bash
python -m template_cloner.cli extract -o <filename.yaml>
```

| Ключ | Описание |
|------|----------|
| `-o, --output` | Имя выходного файла (по умолчанию `template.yaml`) |

**Порядок действий:**
1. Откройте плату в KiCad.
2. Выделите мышью нужные объекты (компоненты, via, дорожки).
3. Выполните команду в терминале.
4. В текущей папке появится YAML-файл с шаблоном.

---

### Команда `place` – применение шаблона

```bash
python -m template_cloner.cli place -i <template.yaml> --origin-x <X> --origin-y <Y>
```

| Ключ | Описание |
|------|----------|
| `-i, --input`  | Файл шаблона (по умолчанию `template.yaml`) |
| `--origin-x`   | Новая координата X центра масс блока (в мм) |
| `--origin-y`   | Новая координата Y центра масс блока (в мм) |

**Порядок действий:**
1. Убедитесь, что в плате присутствуют компоненты с теми же `refdes`, что и в шаблоне (иначе они будут пропущены).
2. Выполните команду – компоненты переместятся, а via и дорожки будут созданы заново в новых позициях.
3. Изменения автоматически фиксируются в плате.

---

## Пример

1. Выделите на плате блок конденсаторов и ферритовый фильтр вместе с дорожками.
2. Сохраните шаблон:
   ```bash
   python -m template_cloner.cli extract -o my_block.yaml
   ```
3. Перенесите блок в точку (120.0, 50.0):
   ```bash
   python -m template_cloner.cli place -i my_block.yaml --origin-x 120.0 --origin-y 50.0
   ```

## Архитектура проекта

```
KiCadTemplateCloner/
├── template_cloner/
│   ├── __init__.py
│   ├── cli.py                # точка входа (команды)
│   ├── extractor.py          # извлечение шаблона
│   ├── applier.py            # применение шаблона
│   ├── template.py           # модели данных (Template, ComponentItem, ViaItem, TrackItem)
│   └── kicad/                # (скопировано из KiCadDecapPlacer) – IPC-взаимодействие
├── tests/                    # тестовые скрипты (не обязательны)
├── requirements.txt
└── README.md
```

Модули:

- **cli.py** – использует `click` для парсинга команд и вызывает соответствующие функции.
- **extractor.py** – подключается к KiCad через `kipy`, получает выделенные объекты, вычисляет центр масс, преобразует координаты в относительные и сохраняет в YAML.
- **applier.py** – загружает YAML, вычисляет новые абсолютные координаты и перемещает/создаёт объекты через `kipy`.
- **template.py** – определяет структуру данных для YAML и методы сохранения/загрузки.
- **kicad/** – содержит адаптер для работы с IPC-соединением (взятый из проверенного проекта `KiCadDecapPlacer`). Обеспечивает стабильное соединение, логирование и базовые операции.

## Как мы получаем объекты и цепи

### Выделенные объекты

1. **Через `board.get_selection()`** получаем все выделенные элементы (объекты могут быть сгруппированы в `Group`).
2. Из каждого элемента извлекаем UUID (`item.id.value`). Если это группа – рекурсивно обходим `item.proto.items`.
3. По UUID находим полные объекты через `board.get_footprints()`, `board.get_vias()`, `board.get_tracks()`.

### Компоненты (Footprints)

- **Reference** → `fp.reference_field.text.value`
- **Value** → `fp.value_field.text.value`
- **Footprint name** → `fp.definition.id`
- **Позиция** → `fp.position.x` и `fp.position.y` (в нанометрах, делим на 1_000_000 для мм)
- **Слой** → `fp.get_layer().name` (или `fp.layer`, если есть)
- **Угол** → `fp.orientation.as_degrees()` (хранится в радианах, преобразуем в градусы)

### Переходные отверстия (Vias)

- **Позиция** → `via.position.x / 1_000_000`
- **Диаметр** → `via.diameter` (в нанометрах)
- **Сверло** → `via.drill.diameter`
- **Цепь** → извлекаем из `via.net.name` (или `via.proto.net.name`)
- **Слой** – обычно не хранится отдельно для via, но можно получить из `via.layer`, если есть.

### Дорожки (Tracks)

- **Начало/конец** → `tr.start` и `tr.end` (объекты с полями `x`, `y` в нанометрах)
- **Ширина** → `tr.width`
- **Цепь** → `tr.net.name`
- **Слой** → `tr.layer`

### Цепи

В некоторых версиях `kipy` невозможно получить цепи для компонентов напрямую через `parent_id`. Мы применили **геометрический маппинг**:
- Строим карту всех футпринтов по их UUID с координатами центра.
- Для каждого пада (`pad`) находим ближайший футпринт по евклидову расстоянию.
- Приписываем имя цепи пада этому футпринту.

Этот метод надёжно работает, даже если API не предоставляет прямой связи.

## Ограничения и дальнейшее развитие

- **Текущая версия** поддерживает клонирование компонентов, via и дорожек. Создание объектов через `board.create_items` может отличаться в разных версиях `kipy` – при необходимости адаптируйте под свой API.
- **Коллизии** не проверяются – новые объекты могут перекрыть существующие.
- **Привязка дорожек к падам** после перемещения не гарантируется – они создаются как отдельные отрезки и могут не совпадать с контактами компонентов. В будущем можно добавить автоматическое переподключение к ближайшим падам по цепи.
