## 🔌 Использованные API KiCad (kipy)

Проект работает через IPC-библиотеку `kipy`, которая обеспечивает прямое соединение с открытой платой в KiCad. Ниже перечислены основные методы, которые мы используем.

### Подключение и получение платы
- **`kicad = kipy.KiCad()`** – создаёт экземпляр клиента IPC.  
- **`board = kicad.get_board()`** – возвращает объект текущей платы.  

### Работа с выделением
- **`board.get_selection()`** – возвращает список всех объектов, выделенных в данный момент в редакторе.  
- У объектов есть **`id`** (с внутренним полем `value`), по которому можно идентифицировать их уникально.  
- Если объект является **группой** (`Group`), его содержимое доступно через `item.proto.items`.  

### Получение всех объектов по типу
- **`board.get_footprints()`** – возвращает список всех компонентов на плате.  
- **`board.get_vias()`** – возвращает список всех переходных отверстий.  
- **`board.get_tracks()`** – возвращает список всех дорожек.  

### Работа с компонентами (`Footprint`)
- **`fp.reference_field.text.value`** – позиционное обозначение (refdes).  
- **`fp.value_field.text.value`** – номинал (value).  
- **`fp.definition.id`** – имя типоразмера (footprint name).  
- **`fp.position`** – объект с полями `x` и `y` (в нанометрах).  
- **`fp.get_layer()`** – возвращает объект слоя; его имя можно получить через `.name`.  
- **`fp.orientation`** – угол поворота (в радианах); метод `as_degrees()` преобразует в градусы.  

### Работа с переходными отверстиями (`Via`)
- **`via.position`** – координаты центра (нм).  
- **`via.diameter`** – внешний диаметр (нм).  
- **`via.drill.diameter`** – диаметр сверла (нм).  
- **`via.net`** – объект сети, его имя через `.name`.  
- Слой не всегда явно задан для via; обычно они сквозные.

### Работа с дорожками (`Track`)
- **`tr.start`** и **`tr.end`** – точки с полями `x`, `y` (нм).  
- **`tr.width`** – ширина дорожки (нм).  
- **`tr.net`** – сеть (аналогично via).  
- **`tr.layer`** – слой (может быть числом или строкой).

### Извлечение цепей (геометрический маппинг)
Поскольку у компонентов нет прямого доступа к своим падам в некоторых версиях `kipy`, мы используем **геометрический маппинг**:
1. Берём все пады платы через **`board.get_pads()`** (если доступен).  
2. Для каждого пада получаем его позицию и имя сети (`pad.net.name`).  
3. Находим ближайший компонент по евклидову расстоянию между позицией пада и центром компонента (из `fp.position`).  
4. Приписываем имя сети этому компоненту.  

Этот метод надёжен и не требует прямой связи между падом и компонентом через ID.

### Создание новых объектов
Для создания via и дорожек мы используем **`board.create_items([...])`**, передавая список объектов с заполненными полями:
```python
via = kipy.Via()
via.position.x = x_nm
via.position.y = y_nm
via.drill.diameter = drill_nm
via.diameter = diam_nm
via.net.name = net_name
board.create_items([via])
```
Аналогично для дорожек:
```python
tr = kipy.Track()
tr.start.x = sx_nm
tr.start.y = sy_nm
tr.end.x = ex_nm
tr.end.y = ey_nm
tr.width = width_nm
tr.net.name = net_name
board.create_items([tr])
```

### Фиксация изменений
- **`board.push_commit()`** – применяет все внесённые изменения и делает их постоянными в открытой плате.
